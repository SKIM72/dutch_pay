# 데이트 비용 정산 웹 애플리케이션 개발

## 1. 프로젝트 개요

여자친구와의 데이트 비용을 효율적으로 정산하고 기록하기 위한 웹 기반 애플리케이션입니다. 엔화(JPY)를 기준으로 정산하며, 다양한 지출 시나리오에 맞춰 공정하게 비용을 분담하고 최종 상계 처리된 금액을 제시합니다. 모든 기록은 데이터베이스에 저장되어 언제든지 조회 및 수정이 가능합니다.

## 2. 주요 기능

- **정산 건별 관리 (날짜 기준):** 날짜를 기준으로 '2026-02-17 도쿄 여행'과 같이 정산 건을 생성하고 관리합니다.
- **유연한 비용 분담:**
    - 1/N 정산
    - 비율(%) 정산 (예: 60% / 40%)
    - 특정 금액 할당 (예: A: 20,000원, B: 30,000원)
- **실시간 환율 적용:** 원화(KRW)로 입력된 금액은 정산 당일 환율을 자동으로 조회하여 기준 통화인 엔화(JPY)로 변환합니다.
- **데이터 관리:**
    - 필수 입력: 메뉴명(결제처), 금액, 결제수단
    - 모든 정산 기록은 Supabase 데이터베이스에 저장됩니다.
    - 과거 기록 조회 및 수정이 가능합니다.
- **최종 정산:** 해당 정산 건에 속한 모든 지출 내역을 합산하고 각자 부담한 금액을 계산하여, 최종적으로 누가 누구에게 얼마를 보내야 하는지 명확하게 알려줍니다.
- **다국어 지원:** 한국어(ko)와 일본어(ja)를 지원하며, 사용자가 언어를 선택할 수 있습니다.
- **반응형 웹 디자인:** PC와 모바일 등 다양한 디바이스에서 최적화된 화면을 제공합니다.

## 3. 개발 계획

### 1단계: 기본 구조 및 화면 설계 (현재 진행중)
- **세부 작업:**
    - `blueprint.md`: 프로젝트 계획서 작성
    - `index.html`: 기본 HTML 구조 설계 (정산 건 선택, 지출 내역 표시, 입력 폼, 정산 결과)
    - `style.css`: 기본 스타일 및 레이아웃 설정
    - `main.js`: 기본 JavaScript 파일 생성

### 2단계: 핵심 로직 구현 (Client-side)
- **세부 작업:**
    - 지출 내역 추가, 삭제 기능
    - 다양한 분담 방식(1/N, 비율, 금액)에 따른 계산 로직 구현
    - 최종 정산 금액 계산 로직 구현

### 3단계: Supabase 연동
- **세부 작업:**
    - Supabase 프로젝트 생성 및 테이블 설계
    - 데이터 조회, 추가, 수정, 삭제(CRUD) 기능 연동

### 4단계: 고급 기능 구현
- **세부 작업:**
    - 환율 정보 API 연동 및 자동 변환 기능
    - 다국어(i18n) 기능 구현

### 5단계: 디자인 및 사용성 개선
- **세부 작업:**
    - CSS를 이용한 시각적 완성도 향상
    - 모바일 반응형 디자인 최종 점검
    - 사용자 피드백을 통한 개선
