# 데이트 비용 정산 웹 애플리케이션 개발

## 1. 프로젝트 개요

커플, 친구 등 두 명의 사용자가 발생하는 비용을 효율적으로 정산하고 기록하기 위한 웹 기반 애플리케이션입니다. 엔화(JPY)를 기준으로 정산하며, 다양한 지출 시나리오에 맞춰 공정하게 비용을 분담하고 최종 상계 처리된 금액을 제시합니다. 모든 기록은 데이터베이스에 저장되어 언제든지 조회 및 수정이 가능합니다.

## 2. 주요 기능

- **정산 건별 관리:**
    - 날짜를 기준으로 '2026-02-17 도쿄 여행'과 같이 정산 건을 생성하고 관리합니다.
    - **각 정산 건마다 참여자 이름(예: 철수, 영희)을 자유롭게 지정할 수 있습니다.**
    - **정산 건별 삭제 기능을 제공하며, 삭제 전 확인 절차를 거칩니다.**
- **유연한 비용 분담:**
    - **1/N 정산:** 총액을 인원수로 동일하게 나눕니다.
    - **금액 직접 입력:** 각 참여자가 부담할 금액을 직접 입력하면, 총액이 자동 계산됩니다.
- **정산 상태 관리:**
    - **'정산 중'과 '정산 완료' 상태를 관리합니다.**
    - 사용자가 '정산 완료하기' 버튼을 눌러야 최종 정산 결과가 표시됩니다.
    - 정산 완료 후 지출 내역이 변경되면 다시 '정산 중' 상태로 전환됩니다.
- **데이터 관리:**
    - 상세 지출 내역을 엑셀과 유사한 테이블 형태로 관리합니다.
    - **상세 지출 내역 개별 삭제 기능을 제공합니다.**
    - 상세 지출 내역 **엑셀(.xlsx) 다운로드 기능을 제공**합니다.
    - 모든 정산 기록은 Supabase 데이터베이스에 저장됩니다.
- **사용자 경험(UX) 개선:**
    - 모든 금액 입력 필드에 쉼표(,) 구분 기호를 자동으로 적용하여 가독성을 높입니다.
    - 다국어 지원 (한국어, 영어, 일본어) 및 브라우저 언어 자동 감지 기능을 제공합니다.
- **반응형 웹 디자인:** PC와 모바일 등 다양한 디바이스에서 최적화된 화면을 제공합니다.

## 3. 개발 계획

### 1단계: 기본 구조 및 화면 설계 (완료)
- `blueprint.md`, `index.html`, `style.css`, `main.js` 기본 구조 완성

### 2단계: 핵심 로직 구현 (완료)
- 지출 내역 추가, 분담 방식 계산 로직 구현
- 지출 내역 테이블 UI 및 엑셀 다운로드 기능 구현
- 정산 건별 참여자 이름 커스터마이징 기능 구현

### 3단계: 고급 기능 및 UX 개선 (현재 진행중)
- **다국어(i18n) 기능 구현**
- **상세 지출 내역 및 정산 건별 삭제 기능 구현**
- **분담 방식 옵션(1/N, 금액 직접 입력) 재구현 및 UX 개선**
- **금액 입력 필드 자동 서식(쉼표) 기능 추가**
- **정산 완료 상태 관리 및 버튼 로직 구현**

### 4단계: Supabase 연동
- Supabase 프로젝트 생성 및 테이블 설계
- 데이터 조회, 추가, 수정, 삭제(CRUD) 기능 연동

### 5단계: 최종 디자인 및 사용성 개선
- CSS를 이용한 시각적 완성도 향상 및 사용자 피드백 반영
- 앱 로딩 시 오늘 날짜(일본 기준) 자동 설정으로 사용자 편의성 증대
